2. Primo avvio

  1. Apri il sito pubblicato oppure avvia npm run serve in locale e visita l'indirizzo indicato.
  2. Al primo accesso inserisci nome utente ed email di recupero: il form non contiene dati personali precompilati.
  3. Scegli una password di almeno 6 caratteri. La password non viene salvata in chiaro.
  4. Dopo la creazione dell'account, scegli uno dei profili disponibili.

La password e i dati dell'account locale restano nel browser del dispositivo, dentro lo storage del sito.

3. Profili familiari

L'account e unico, ma i dati dei profili sono distinti. Ogni profilo ha cronologia, watchlist, preferiti, rating, statistiche e preferenze separate.

Profilo Daniela

Destinazione consigliata per importare la libreria TV Time.

Profilo Elena

Parte senza dati demo e puo essere rinominato dalle impostazioni.

4. Importare lo ZIP GDPR di TV Time

  1. Accedi al profilo in cui vuoi importare la libreria.
  2. Apri Profilo - Importa ed esporta.
  3. Seleziona direttamente gdpr-data.zip. Non serve estrarlo.
  4. Controlla l'anteprima, poi avvia l'importazione.
  5. Al termine verifica serie, film, preferiti, rating e statistiche.
Lo ZIP GDPR viene letto localmente nel browser. I file tecnici sensibili vengono esclusi dall'importatore.

6. Backup ed esportazioni

Dopo l'importazione crea subito un backup completo. I file sono salvati nella cartella Download del dispositivo.

  • Backup ZIP: formato migliore per ripristinare Watchverse.
  • CSV interoperabile: cronologia leggibile con film, episodi, date e rating.
  • JSON pronto per Trakt: export dedicato alla riconciliazione esterna.

7. Pubblicazione online

La pubblicazione GitHub Pages usa la build generata da npm run build. La sincronizzazione cloud richiede una configurazione Supabase separata.

  1. Inserisci in config.js solo URL Supabase, chiave publishable e URL del proxy TMDB.
  2. Non inserire mai nel client la chiave service_role o il token TMDB.
  3. Non pubblicare backup personali o ZIP GDPR originali.

8. Privacy e file da non pubblicare

Lo ZIP GDPR originale e sensibile.

  • Conservalo in una cartella privata.
  • Non caricarlo su repository GitHub o hosting pubblico.
  • Non condividere pacchetti personali con cronologia o backup privati.
  • Usa soltanto il pacchetto pubblico privo di cronologia personale.
